# 基于模板图片创建地图的工作流（背景 + 地面/空中块皮肤）

这份 Note 对应当前工程的“每张地图从创建之初就绑定一张 png/jpg/bmp 模板图”的做法。

## 核心机制（已实现）

- 地图脚本：[TemplateRoomMap.gd](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Scripts/World/TemplateRoomMap.gd)
- 可复制的地图模板场景：[room_template_from_image.tscn](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Maps/room_template_from_image.tscn)

你给每张地图准备一张“模板图片”，脚本会做两件事：

1. **背景绑定**：`background_texture`（优先）或 `template` 会自动显示为全屏背景。
2. **地面/空中石块绑定皮肤**：从 `template` 里按颜色识别 tile，生成到 `TileMap/Foreground`。

## 模板图片的约定（非常简单）

建议两种尺寸（二选一）：

1. **27×15 像素**：1 像素 = 1 个 tile（最清晰、最不费资源）
2. **864×480 像素**：刚好等于 1 个房间屏幕（脚本会按网格采样到 27×15）

颜色约定（可改，但先按这个跑通）：

- `ground_color = #FFFFFF`：地面 tile
- `air_block_color = #808080`：空中石块 tile
- 透明（alpha ≤ `empty_alpha_threshold`）：空

## 新建一张“绑定模板”的地图（推荐流程）

1. 复制 [room_template_from_image.tscn](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Maps/room_template_from_image.tscn)
   - 重命名为你的地图，如：`map_starsky.tscn`、`map_forest_night.tscn`
2. 在 Godot Inspector 里选中根节点 `Map`，设置：
   - `template`：选择你的模板 png/jpg/bmp
   - `background_texture`（可选）：如果背景要用另一张图，就填它；否则留空，默认用 `template` 当背景
3. 设置“皮肤”（每张地图可不同）：
   - 地面皮肤：`ground_source_id` + `ground_atlas_coords`
   - 空中石块皮肤：`air_source_id` + `air_atlas_coords`
4. 设置出口（左右出口就是两个 Portal 实例）：
   - `ExitLeft.target_area / target_map`
   - `ExitRight.target_area / target_map`

## 重要：让 MetSys 识别到这张新房间（否则运行会报错）

MetroidvaniaSystem 的房间必须在 `MapData.txt` 里有 cell 绑定，否则 `RoomInstance` 会拿不到 cells/room_id。

- 文件位置：`res://CoreEngine/Maps/MapData.txt`
- 你需要添加一段：

```
[x,y,layer]
0,0,0,0|||uid://你的地图scene_uid
```

提示：
- `uid://...` 来自你的 `.tscn` 文件第一行的 `uid="uid://..."`。
- `x,y,layer` 选一个不冲突的坐标即可（工程里现有地图用的是 0 附近，你可以从更远处开始，比如 (20,0,0)）。
