## 2. 地图/房间系统（MetroidvaniaSystem）

核心能力来自 MetroidvaniaSystem 插件（`MetSys` 单例）：

- 房间加载与切换
  - `Game` 继承 `MetSysGame`，管理当前 `map`（房间场景）与 `player`（见 [Game.gd](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Scripts/Systems/Game.gd)）
  - 启用 `RoomTransitions` 模块：监听 `MetSys.room_changed`，切换房间并自动修正玩家入口偏移（见 [RoomTransitions.gd](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/addons/MetroidvaniaSystem/Template/Scripts/Modules/RoomTransitions.gd)）
- 小地图与大地图窗口
  - 小地图：`addons/MetroidvaniaSystem/Template/Nodes/Minimap.tscn` 实例挂在 `UI/Minimap`
  - 大地图窗口：`Game.tscn` 内嵌 `MapWindow` 脚本（按 `M` 打开/关闭、按方向键平移、按 `D` 显示“起点→当前位置”向量）（见 [Game.tscn](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Game.tscn#L12-L116)）
  - 地图完成度百分比：监听 `MetSys.map_updated` 刷新（同上 `PercentLabel` 脚本）
- 可存储对象（storable objects）与地图标记
  - 收集品在 `_ready` 时注册 marker，收集后通过 `MetSys.store_object(self)` 存储，从而实现“收集状态持久化 + 地图标记更新”（见 [Collectible.tscn](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Objects/Collectible.tscn#L6-L33)）

## 3. 玩家控制与能力系统

玩家脚本：见 [Player.gd](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Scripts/Characters/Player.gd)

- 输入映射（运行时确保存在）
  - 移动：A/D；下蹲/下：S；跳：Space/K；攻击：J；瞄准上：W；背包：X；装备页：Z
- 移动/跳跃
  - 带加速的横向移动（`SPEED_MIN/MAX + ACCEL`）
  - Coyote Time（离地短时间仍可跳）
  - Short Hop（松开跳跃键提前截断上升速度）
  - 双跳能力：`&"double_jump" in abilities` 时启用（能力集合被纳入存档，见第 7 节）
- 射击（子弹）
  - `attack` 触发，带冷却 `shoot_cooldown_seconds`
  - 发射方向由 `FireAngleRule` 计算：同时按上下/左右时支持更自然的 8 向与偏转（见 [FireAngleRule.gd](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Scripts/Input/FireAngleRule.gd)）
- 防御（格挡）
  - “锁定方向（W 或 S）+ 反向键”进入防御状态，并调用 Combatant 的 `set_blocking(true/false)`
- 死亡/重生（回到进入房间时记录的位置）
  - `on_enter()` 记录 `reset_position`
  - 死亡后发出战斗结果提示，再触发 `kill()`：请求重载当前房间（见第 5/6 节的消息流）

## 4. 战斗系统（HP/护甲/装备/伤害）

战斗组件：见 [Combatant.gd](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Scripts/Combat/Combatant.gd)

- 角色属性
  - `max_hp/hp/base_armor`，支持装备叠加护甲
  - 支持盾牌的“格挡护甲”（blocking 且正面受击时额外叠加）
- 伤害结算
  - 实际伤害：`raw_damage * 100 / (100 + armor)`
  - `health_changed`、`equipment_changed`、`died` 信号驱动 UI/逻辑
- 投射物（子弹）系统
  - 子弹为 `Area2D`，按 `source_faction` 只伤害对应阵营（player/enemy 分组）（见 [Bullet.gd](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Scripts/World/Bullet.gd)）
  - 命中目标时优先调用 `apply_hit/apply_raw_damage`，否则尝试找子节点 `Combatant`

敌人脚本：见 [AliceNPC.gd](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Scripts/Characters/AliceNPC.gd)

## 11. 数值系统与结算流水线（Value + Settlement）

- 值存储：`ValueStore`（key→value），对外以 Workbench 服务 `"values"` 暴露
- 交易消息：`value_tx`（key + delta + context），会被 `SettlementEngine` 入队并在每帧 `flush()` 统一结算
- 已接入的示例值：`collectibles`
  - 收集品发送 `value_tx` 增加 `collectibles`
  - `ValueSettlementActor` 监听 value 变化，把 `collectibles` 同步回 `Game.collectibles`（并进一步同步 UI/Workbench）（见 [ValueSettlementActor.gd](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Scripts/Actor/ValueSettlementActor.gd)）

## 12. 程序化生成房间（DiceRoom 随机层）

触发点：`DiceRoom.tscn` 的 Dice Area2D（见 [DiceRoom.tscn](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Maps/DiceRoom.tscn#L8-L163)）

- 生成方式：Thread 内构建“逻辑网格”，再用 `MetSys.get_map_builder()` 创建 cell、设置 borders，并为 cell 分配一个“伪路径”
- 伪路径格式：`GEN/<id>/<exitsBitmask>/<has_collectible>`，用于让 `Game._load_room()` 识别并加载“原型 Junction 场景 + 应用配置”
- 生成结果持久化：把生成房间的坐标列表写入 `ProgressData.generated_rooms`，从而支持存档与再次进入时的状态恢复
- “进入随机层”事件：`Game.init_room()` 通过 `map.scene_file_path` 是否以 `GEN` 开头判断进入/退出随机层，并发出 `level_event_request`，最终由 `PlayerDataManagerActor` 应用倍率（见 [Game.gd](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Scripts/Systems/Game.gd#L343-L353)）

## 13. 传送与循环房间（Portal / LoopScript）

- Portal：玩家进入后 tween 吸入传送门，再发出 `load_room_request`，并延迟清理玩家 event（见 [Portal.gd](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Scripts/World/Portal.gd)）
- LoopScript：在过渡区劫持房间切换，支持两种模式
  - Teleport：监听 `MetSys.cell_changed`，主动请求 load_room + shift_player
  - Replace：监听 `MetSys.room_changed`，清理当前 room 实例以阻止自动位移，并手动 shift_player（见 [LoopScript.gd](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Scripts/World/LoopScript.gd)）
