# 2026-01-31 游戏状态管理设计

## 现状

- 当前项目核心是随机地图 + 跑图 Demo
- 还没有剧情流程/章节推进等“叙事状态”

## 目标（最低程度）

- 引入“幕”（序幕、第一幕、第二幕……）作为游戏进行阶段的状态
- 幕切换时给玩家一个文字提示：例如“进入第一幕”
- 后续可以把幕用于控制更深层的内容（例如不同的背景/氛围/玩法开关），但当前先不实现这些效果

## 设计方案

### 状态数据

- 使用 `Game.gd` 内的 `story_act: int` 存储当前幕
- `0` 表示“序幕”，`1` 表示“第一幕”，以此类推
- 将 `story_act` 写入存档字典，随存档一起保存/读取

代码位置：

### 状态切换 API

- `set_story_act(new_act, show_message=true)`：设置为指定幕（可选显示提示）
- `advance_story_act()`：推进到下一幕并提示

### UI 提示（Toast）

- 在 [Game.tscn](file:///d:/Task_Panel/0_AliceJOH/0_AOJ_Reference/workspace/Metroidvania-System-master/CoreEngine/Game.tscn) 的 `UI` 下新增 `StoryToast` Label
- 切换幕时显示 “进入{幕名}”，延时后淡出隐藏

### 当前触发方式（临时）

- 为了快速验证流程，目前用 `F3` 往工作台写入 `story_event/debug_next_act`，由幕管理逻辑计算后切到下一幕

## 分支管理（Workplace 驱动）

### 基本思路

- 游戏内所有“会影响剧情分支”的事情，都以事件形式写入公共工作台（Workbench）
- 幕管理 Actor 订阅这些事件，基于事件与内部事实（已进过哪些房间、拿过哪些道具、杀过哪些 NPC、哪些 flag 置位等）计算下一幕
- 计算结果以 `story_act_set` 的形式回写到工作台，由 Game 接收并替换当前幕，同时给玩家提示

### 事件格式（当前约定）

- 写入事件：
  - `{"type": &"story_event", "name": &"map_entered", "room": "<room_id>"}`
  - `{"type": &"story_event", "name": &"item_acquired", "item": "<item_id>"}`
  - `{"type": &"story_event", "name": &"npc_killed", "npc": "<npc_id>"}`
  - `{"type": &"story_event", "name": &"flag_set", "flag": "<flag_id>"}`
  - `{"type": &"story_event", "name": &"prologue_end"}`

- 幕管理回写：
  - `{"type": &"story_act_set", "value": <int>, "reason": <StringName>}`

### 当前落地情况

- `Game.init_room()` 会自动写入 `map_entered`（每次房间加载完成后）
- `F3` 写入 `debug_next_act`（用于临时调试推进）
- 幕管理逻辑目前以 `Game.gd` 内的 `StoryActManager` 类实现（注册为 Workbench actor），规则表可按需扩展
